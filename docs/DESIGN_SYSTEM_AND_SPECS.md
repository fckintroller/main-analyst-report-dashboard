# 🎨 KOREAN ANALYST REPORT: Design System & Functional Specifications

## 1. Design System (디자인 시스템)

### 1.1. Color Palette (색상 팔레트)
대시보드는 눈부심을 방지하고 데이터 가독성을 극대화하기 위해 **'Dark Pine Green'** 테마를 지향합니다.

**기본 테마 색상 (`:root` 변수)**
- `--bg-color`: `#040608` (초저조도 딥 블랙 - 전체 배경)
- `--card-bg`: `#080c12` (다크 파인 차콜 - 카드 및 주요 섹션 배경)
- `--card-border`: `#101620` (기본 테두리 색상)
- `--text-main`: `#d1d5db` (회백색 - 주 본문 텍스트)
- `--text-sub`: `#6b7280` (올리브 그레이 - 보조 설명 텍스트)
- `--primary`: `#047857` (딥 파인 그린 - 포인트 강조 및 배지 배경)
- `--upgrade`: `#059669` (에메랄드 그린 - 상향/긍정 지표)
- `--downgrade`: `#b91c1c` (다크 레드 - 하향/부정 지표)

## 2. Functional Specifications (기능 명세서)

### 2.1. 결합 섹터별 애널리스트 (Tab 1)
- **데이터 소스**: `data/analyst_database.json` -> `analysts`
- **필터링 로직**: `merged_sector`를 기준으로 필터 버튼을 동적 생성. `&` 등 특수문자 충돌 방지를 위해 `addEventListener`를 통해 원본 문자열 직접 매칭.
- **인터랙션**: 마우스 오버 시 카드가 위로 3px 이동하며 메인 테마색(`--primary`)의 글로우 효과가 나타나야 함.

### 2.2. 투자의견 변동 알림 (Tab 2)
- **데이터 소스**: `data/analyst_database.json` -> `recommendations`
- **핵심 UI**: 
  - 가로 스크롤이 가능한 반응형 테이블 구조.
  - 변동 흐름(Previous ➔ Current)을 시각적 화살표와 각 등급 색상으로 표시.
  - 리서치 코멘트는 가독성을 위해 기울임꼴(`italic`)과 줄바꿈 허용.

### 2.3. 리서치 보고서 (Tab 3)
- **데이터 소스**: `data/analyst_database.json` -> `reports`
- **UI 구조**: 카드 헤더(제목/작성자), 본문(요약), 푸터(종목/의견/목표가)로 구성된 3단 그리드 레이아웃.

### 2.4. 주가 & 코스피 비교 분석 차트 (Tab 4) - **Core Feature**
- **기술 스택**: Chart.js v4 + `chartjs-plugin-annotation`
- **Y축 이중 축(Dual-Axis) 규칙**:
  - **수수익률 모드**: 좌측 단일 축(%) 표시.
  - **가격 모드**: 종목(좌측: 원)과 코스피(우측: pt)를 별도 축으로 자동 분리하여 렌더링.
- **이벤트 시각화**:
  - 오늘을 기준으로 과거 일정(회색 점선)과 미래 일정(노란색 점선)을 차트에 세로선으로 표시.
  - 차트 세로선 마우스 오버 시 **하단 이벤트 리스트 자동 하이라이트(흰색 배경/빨간 글씨)** 및 스크롤 연동.
- **미래 예측**: 차트 우측에 30일간의 빈 공간(Padding)을 두어 예정된 일정을 차트 위에 미리 표시 가능하도록 함.

### 2.5. 경제 캘린더 (Tab 5)
- **그룹화**: [과거 / 지난주 / 오늘 / 내일 / 이번 주 / 다음 주 / 이번 달]로 자동 분류.
- **상태 표시**: 
  - 과거 일정은 기본적으로 접혀 있음 (`<details>`).
  - **High Impact**: 중요도가 'High'인 이벤트는 노란색 강조 선과 틴트 배경색 적용.

## 3. Architecture & Security Rules (개발 및 보안 규칙)

### 3.1. 보안 강화
- **XSS 필터링**: 모든 웹 렌더링 텍스트는 반드시 `escapeHTML()`을 거쳐야 함. 특히 크롤링된 데이터(제목, 요약 등)는 엄격히 적용.
- **안전한 파싱**: Python 스크립트에서 JSON 데이터를 다룰 때 `ast.literal_eval` 대신 표준 `json.loads`와 데이터 정제 로직을 병행 사용.

### 3.2. 데이터 관리
- **중복 방지**: 리서치 리포트 수집 시 `(날짜 + 종목명 + 제목)` 복합 키를 생성하여 중복 저장을 방지함.
- **경로 관리**: 모든 파일 경로는 `scripts/utils.py`에서 중앙 집중식으로 관리하며, 루트 경로 기반의 상대 경로를 사용하여 폴더 이동에도 유연하게 대응함.
